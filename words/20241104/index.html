Tana 的命令功能很有意思，我也花了一段时间，翻阅 Slack，借鉴别人的方法，才逐渐搭建了对我来说好用的命令。偶尔会觉得它和快捷指令一样有意思。总之，今天就来开箱，给大家品鉴一下我常用好用的 Tana Command。

读者仅需了解超标签、字段、节点显示区别和命令节点的创建方法。

一键整理 Inbox 节点的命令

功能

将子节点打上标签，移动到创建日期节点。

使用场景

Raycast 和 Tana Capture 的笔记直接放进 Inbox ，一键整理后，方便我做每周和每日回顾。

调用方式

在 INBOX 节点上直接显示命令图标，点击调用：命令面板 ➡️ Configure Node ➡️ Command 字段：引用该命令节点链接。

点击打开节点链接的命令

使用场景&功能

有一些文章可以直接打开链接查看原文，标题节点直达原文，无需查看节点内部 URL 节点。

调用方式

URL 字段不为空的节点（通过Node filter 字段筛选）都有这个命令图标，一键调用。

点击更新任务状态的命令

使用场景&功能

更新写作草稿_状态_（选择字段）为“已完成”。

调用方式

published? 字段值为“not yet”的节点（通过Node filter 字段筛选）都有这个命令图标，一键调用。
